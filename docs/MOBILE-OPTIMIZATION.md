# 移动端响应式优化说明

## 优化目标

针对手机显示进行专门优化，确保在所有移动设备和 PC 浏览器上都有良好的显示效果。

## 主要优化内容

### 1. **Header 组件** ([src/components/Header.vue](src/components/Header.vue))

**移动端优化：**
- ✅ 减小内边距：`px-6 py-6` → 移动端 `px-4 py-5`
- ✅ 减小图标容器：`w-14 h-14` → 移动端 `w-12 h-12`
- ✅ 减小图标尺寸：`w-7 h-7` → 移动端 `w-6 h-6`
- ✅ 调整标题大小：`text-3xl` → 移动端 `text-2xl`
- ✅ 调整副标题：`text-sm` → 移动端 `text-xs`
- ✅ 减小间距：`mb-8` → 移动端 `mb-6`

**效果：**
- 移动端高度减少约 30%
- 更紧凑，不占用过多屏幕空间

### 2. **App.vue 整体布局** ([src/App.vue](src/App.vue))

**移动端优化：**
- ✅ 减小外边距：`p-4` → 移动端 `p-3`
- ✅ 加载状态容器：`w-16 h-16` → 移动端 `w-14 h-14`
- ✅ 加载文字：`text-base` → 移动端 `text-sm`
- ✅ 免责声明间距：`mt-12` → 移动端 `mt-8`
- ✅ 免责声明内边距：`px-5 py-4` → 移动端 `px-4 py-3`
- ✅ 图标容器：`w-5 h-5` → 移动端 `w-4 h-4`
- ✅ 文字大小全面优化

**效果：**
- 移动端整体更紧凑
- 减少不必要的空白空间

### 3. **PostCard 组件** ([src/components/PostCard.vue](src/components/PostCard.vue))

**移动端优化：**
- ✅ 卡片间距：`mb-8` → 移动端 `mb-6`
- ✅ 卡片内边距：`p-5` → 移动端 `p-4`
- ✅ 时间标签间距：`gap-1.5` → 移动端 `gap-1`
- ✅ 时间标签内边距：`px-2.5 py-1` → 移动端 `px-2 py-0.5`
- ✅ 时间图标：`w-3.5 h-3.5` → 移动端 `w-3 h-3`
- ✅ 时间文字：`text-xs` → 移动端 `text-[11px]`
- ✅ 内容文字：`text-[15px]` → 移动端 `text-sm`
- ✅ "阅读更多"：`text-xs` → 移动端 `text-[11px]`
- ✅ 时间轴圆点：`12px` → 移动端 `10px`

**效果：**
- 移动端卡片更紧凑
- 文字大小适合小屏幕阅读

### 4. **PostList 组件** ([src/components/PostList.vue](src/components/PostList.vue))

**移动端优化：**
- ✅ 左侧间距：`pl-10` → 移动端 `pl-8`
- ✅ 时间轴线条位置：`left: 16px` → 移动端 `left: 13px`
- ✅ 时间轴线条宽度：`2.5px` → 移动端 `2px`

**效果：**
- 时间轴与圆点完美对齐
- 移动端线条更细，更精致

## 响应式断点

使用 Tailwind CSS 的 `sm:` 断点（640px）：

```css
/* 移动端（< 640px） */
默认样式

/* 桌面端（≥ 640px） */
sm: 前缀的样式
```

## 弹性布局特点

### 1. **流式布局**
- 使用 `max-w-[750px]` 限制最大宽度
- 自动居中对齐
- 两侧自适应边距

### 2. **弹性间距**
- 所有间距使用相对单位
- 移动端自动缩小
- 保持视觉比例

### 3. **弹性字体**
- 标题：`text-3xl` → `text-2xl`
- 正文：`text-[15px]` → `text-sm`
- 小字：`text-xs` → `text-[11px]`

### 4. **弹性组件**
- 图标容器自适应
- 按钮和标签自适应
- 卡片内边距自适应

## 性能优化

### CSS 优化
- ✅ 使用 Tailwind CSS 的 JIT 模式
- ✅ 只生成使用的样式

### 动画优化

### 加载优化
